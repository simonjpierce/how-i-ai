# Lesson 1 — The hook

The fastest way to understand this system is to see what a real session looks like. A short video version will land at the top of this page in time; for now, here's the text walkthrough.

## What it looks like

Imagine you're sitting down on a Tuesday morning. You have to draft the introduction to a funder report by lunchtime. The relevant context is scattered across half a dozen vault files — last week's meeting transcript, the previous interim report, a couple of papers you've been reading, a budget figure from a spreadsheet, a quote from a board member.

You open Claude Code with your Obsidian vault as the working folder. You type (or dictate):

> *"Draft the introduction to the Funder X interim report. The project is the Mozambique whale shark monitoring programme — read the relevant project notes and the previous interim. Match the tone of the previous one. Flag anything that doesn't have evidence in the vault."*

Claude doesn't ask you to paste anything. It searches your vault, opens the project file, reads the last interim, scans two related meeting transcripts, picks up the budget note. It produces a draft in about ninety seconds:

> *"Here's a 280-word introduction. I pulled the achievements summary from your 5 May meeting notes and the budget figure from your Q1 tracker. The previous interim led with the partnership angle, so I followed that pattern. One flag — you reference 'a successful site visit in March' but I can't find a corresponding project note. Want me to search wider, or do you want to add the note?"*

You read the draft. Most of it is fine. You correct two factual details and rewrite one sentence. While Claude waits, you flip to another Obsidian tab and add the missing site-visit note. Then:

> *"Done. Draft the next section: methods and outcomes. Same approach."*

By 10:30am you've drafted the whole report. Tomorrow's session will read the wrap-up handover note Claude wrote at the end and pick up where you left off.

## What just happened

A few things to notice:

- **Claude wasn't briefed.** It didn't get a wall of context pasted in. It read the vault notes itself, pulled what it needed, and worked from there.
- **The "context" was your normal notes.** The same files you write in Obsidian are the files Claude reads. There's no separate "AI context folder" to maintain in parallel.
- **The output landed in the vault.** Whatever Claude drafted is a file you can open in Obsidian, edit, and build on. Tomorrow's session will read it without you having to remember to paste anything.
- **It pushed back.** Claude flagged a missing piece of context rather than fabricating around it. Claude Code is more willing to do this than browser chat — it can see your actual files, so it can't just rubber-stamp ideas.

That's the whole pitch. The rest of the course is how you set this up for yourself.

## If you're a Claude Chat user already

You're using a much-reduced version of the same model. The leap to Claude Code is not "learn a new tool" — it's "give Claude the keys to your files and let the same conversation be useful for longer."

If you've ever:

- Pasted the same context into the top of every Claude conversation
- Lost track of where a useful draft ended up
- Re-explained your project to Claude for the third time this week

…then this is the upgrade.
